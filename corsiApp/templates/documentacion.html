Documentación

Para comenzar:

  • Cargar Protocolo desde archivo Json -> (Ejemplo de protocolo.json)
  • Cargar una Toma de Datos con sus usuarios y asociarla al protocolo -> (Ejemplo de tomaDeDatos.json)
  • Ahora ya es posible Evaluar!
  • Para ver los protocolos cargados y sus tomas de datos asociadas, ir a listado de Protocolos cargados.
  • Para ver los resultados de un Intento, ir a listado de Protocolos, seleccionar la Toma de Datos y para cada uno de los usuarios se listará cada uno de los intentos realizados.

Protocolos

  • Los protocolos se definen a través de un archivo con formato json.
  • Los campos obligatorios son los de nombreProtocolo y trials (secuencias de botones).
  • Junto a cada trial, se define si es de prueba o no. [True/False].
  • Todo lo que no esté definido, se completará con sus valores por defecto, indicados en el Ejemplo de Protocolo Completo.

Ejemplo de Protocolo Mínimo

{
	"protocolo": {
		"nombreProtocolo": "Protocolo Minimo"
	},
	"properties": {
	},
	"trials": {
		"1": ["A","true"],
		"2": ["B","false"]
	}
}
  • Si no se detallan las properties, se completarán por defecto como indica el Ejemplo de Protocolo Completo.

Ejemplo de Protocolo Completo

Descargar ejemplo 
{
	"protocolo": {
		"nombreProtocolo": "Protocolo Ejemplo Completo"
	},
	"properties": {
		"tiempoEncendidoLuz": 750,
		"tiempoEntreLuces": 750,
		"tiempoComienzaIntento": 0,
		"timeoutRespuesta": 35000,
		"cantErroresCorte": 0,
		"reverso": "False",
		"botonRojoIniciaTrial":"False",
		"luzComienzaSecuencia":"True",
		"colorLuzComienzaSecuencia":"blue",
		"tiempoLuzComienzaSecuencia":"500",
		"repeticionLuzComienzaSecuencia":"3",
		"intervaloRepeticionLuzComienzaSecuencia":"100",
		"luzEsperaBotones":"True",
		"colorLuzEsperaBotones":"blue",
		"luzAnalisisIntento":"False",
		"colorLuzAnalisisIntento":"yellow",
		"feedbackLuces": "False",
		"colorSecuenciaCorrecta":"green",
		"colorSecuenciaIncorrecta":"red",
		"tiempoLuzFeedback":"200",
		"repeticionLuzFeedback":"3",
		"intervaloRepeticionLuzFeedback":"100",
		"luzEsperaProxSecuencia": "False",
		"colorLuzEsperaProxSecuencia": "blue",
		"feedbackSonido":"False",
		"luzTerminaTomaDatos":"False"
	},
	"trials": {
		"1": ["GI","true"],
		"2": ["B","false"],
		"3": ["CA","false"],
		"4": ["DGH","false"],
		"5": ["ACD","false"]
	}
}
  • BotonRojoIniciaTrial habilitado permite que el usuario pueda comenzar la siguiente secuencia con el botón rojo. Si está en False lo tiene que hacer el operador mediante el botón "Siguiente Secuencia".
  • LuzEsperaProxSecuencia permite habilitar la luz (y el colorLuzEsperaProxSecuencia) mientras se espera la orden para comenzar una próxima secuencia. En False se mantendrá apagada.
  • El reverso habilitado evalua las secuencias de manera inversa. Ej: Si la secuencia es ABC lo correcto será que el usuario presione CBA.
  • El feedbackLuces, si está desactivado, sólo dará feedback para las secuencias de prueba.
  • El feedbackSonido, si está activado, indicará con tres beeps o uno contínuo si la respuesta ha sido correcta o incorrecta, respectivamente.
  • La luzTerminaTomaDatos indica con verde o rojo si se han completado correctamente o no todos los trials del Protocolo.

Toma De Datos

  • Las tomas de datos definen los usuarios que pueden ser evaluados en un protocolo.
  • Debe tener un nombre y al cargarla se debe asociar sólo a un protocolo.
  • Se pueden agregar usuarios a una toma de datos cargada.
  • La fecha de la toma de datos se completará automáticamente al cargarla.

Ejemplo de Toma de Datos

Descargar ejemplo 
{
    "tomaDeDatos": {
        "nombreTomaDeDatos": "TomaDatosEjemplo"
    },
    "usuarios": {
        "nombres": ["User1","User2","User3"]
    }
}
